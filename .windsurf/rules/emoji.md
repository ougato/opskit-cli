# Emoji 图标规则

## 背景

本项目打包为 Windows 单文件 exe，用户双击运行时宿主终端为 **conhost.exe**（Windows 传统控制台）。conhost 对 emoji 的支持有严格限制，必须遵守以下规则以确保图标在所有环境下**彩色渲染一致**。

## 核心规则

### 必须使用 Supplementary Plane（4 字节）emoji

- Unicode 码点范围：`U+1F000` ~ `U+1FFFF`
- YAML 转义格式：`"\U0001Fxxx"`
- 示例：`"\U0001F4E5"` → 📥、`"\U0001F53C"` → 🔼

### 严格禁止使用 `\uFE0F` 变体选择符

- **禁止** `"\u2B06\uFE0F"`（⬆️）— conhost 下退化为单色 `↑`
- **禁止** `"\u25B6\uFE0F"`（▶️）— conhost 下退化为单色 `▶`
- **禁止** `"\u26A0\uFE0F"`（⚠️）— conhost 下退化为单色 `⚠`
- **禁止** `"\u2139\uFE0F"`（ℹ️）— conhost 下退化为单色 `ℹ`
- **禁止** 任何 `\uXXXX\uFE0F` 格式的组合

### BMP 范围 emoji 的使用限制

以下 BMP 范围字符（`U+2xxx`）在 conhost 下**可能**显示为单色，但目前项目中保留使用：

- `"\u2705"` ✅、`"\u274C"` ❌、`"\u2795"` ➕、`"\u2796"` ➖、`"\u26A1"` ⚡

如发现渲染异常，应替换为 Supplementary Plane 等价图标。

### 新增图标时的检查流程

1. 确认码点在 `U+1F000` ~ `U+1FFFF` 范围内（优先）
2. 确认**不含** `\uFE0F` 变体选择符
3. 在 conhost 下实测彩色渲染（双击 exe 验证）

## 主题文件位置

- `core/themes/catppuccin.yaml` — 唯一主题文件
- 所有图标定义在 `icons:` 节下
- YAML 中使用 `"\U0001Fxxx"` 转义语法
