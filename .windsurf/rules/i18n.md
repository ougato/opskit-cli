# 国际化强制规则（I18n Enforcement Rule）

> **本规则为最高优先级强制规则，任何新增代码必须无条件遵守，无例外。**

---

## 一、核心原则

**项目中所有对外用户可见的文字，必须且只能通过 `t()` 本地化函数输出，严禁任何形式的硬编码。**

---

## 二、强制覆盖范围

以下场景全部强制使用 `t()`，无一例外：

| 场景 | 典型代码位置 |
|---|---|
| 安装/卸载/升级错误消息 | `raise InstallError(...)` / `raise UninstallError(...)` |
| 控制台打印提示 | `console.print(...)` / `print_error(...)` / `print_warning(...)` / `print_success(...)` |
| 菜单标签、子标题 | `select(subtitle=...)` / `choices` 列表 |
| 确认提示 | `confirm(prompt=...)` |
| 文本输入提示 | `text_input(prompt=...)` |
| 进度步骤描述 | `InstallStep(...)` / `sp.step(...)` |
| 异常消息（ValueError / RuntimeError 等可见给用户） | `raise ValueError(t(...))` |
| 任何 f-string 用户文本 | 全部禁止 |

---

## 三、正确写法示例

```python
from core.i18n import t

# ✅ 正确：InstallError 使用 t()
raise InstallError(t("software.golang_error.download_failed", version=version, error=e))

# ✅ 正确：print_error 使用 t()
print_error(t("software.mysql_error.not_installed", version=version))

# ✅ 正确：confirm 使用 t()
confirm(breadcrumb=breadcrumb, prompt=t("deps.confirm_upgrade", key=key, required=required))

# ✅ 正确：进度步骤
InstallStep("software.step.download")

# ✅ 正确：菜单
{"key": "1", "label": f"{get_icon('install')} {t('software.install')}"}
```

---

## 四、禁止写法（违规示例）

```python
# ❌ 禁止：InstallError 硬编码中文
raise InstallError("MySQL 下载失败")

# ❌ 禁止：f-string 硬编码
raise InstallError(f"MySQL {version} 下载失败：{e}")

# ❌ 禁止：print 硬编码
console.print("安装成功")
print_error("版本不支持")

# ❌ 禁止：confirm 硬编码
confirm(prompt="确认继续？")

# ❌ 禁止：ValueError 硬编码
raise ValueError("令牌解码失败")
```

## 五、locale 文件规范

### 文件位置
- 中文：`core/locale/zh.yaml`
- 英文：`core/locale/en.yaml`

### key 命名规则
1. 采用点分层级命名：`模块.子分类.动作`
2. 示例：`software.mysql_error.download_failed`
3. 占位符使用 `{param}` 格式，调用时通过关键字参数传入
4. 各 recipe 错误统一放在 `software.<name>_error` 命名空间下

### 命名空间速查
| 模块 | 命名空间 |
|---|---|
| golang | `software.golang_error.*` |
| nodejs | `software.nodejs_error.*` |
| java | `software.java_error.*` |
| python | `software.python_error.*` |
| mysql | `software.mysql_error.*` |
| postgresql | `software.postgresql_error.*` |
| mongodb | `software.mongodb_error.*` |
| redis | `software.redis_error.*` |
| docker | `software.docker_error.*` |
| nginx | `software.nginx_error.*` |
| wireguard | `wireguard.error.*` |
| 依赖解析 | `deps.*` |

### 新增 key 流程（必须同时做）
1. 在 `core/locale/zh.yaml` 添加中文 key
2. 在 `core/locale/en.yaml` 添加英文 key（结构和 key 名称完全一致）
3. 在代码中通过 `t("key", ...)` 使用

---

## 六、自动检测

运行以下命令验证所有 `t()` 调用的 key 均存在于 zh.yaml：

```bash
uv run python tests/_check_locale_keys.py
```

**每次新增 locale key 后必须运行此检测，确保零遗漏。**

---

## 七、代码审查检查清单

新增或修改涉及用户提示的代码时，必须逐项确认：

- [ ] 所有 `raise InstallError(...)` 使用 `t()`，无 f-string
- [ ] 所有 `raise UninstallError(...)` 使用 `t()`，无 f-string
- [ ] 所有 `raise ValueError(...)` 若可能展示给用户，使用 `t()`
- [ ] 所有 `print_error / print_warning / print_success` 使用 `t()`
- [ ] 所有 `console.print(...)` 中的文字部分使用 `t()`
- [ ] 所有 `confirm(prompt=...)` 使用 `t()`
- [ ] 所有 `text_input(prompt=...)` 使用 `t()`
- [ ] 新增 key 已同时写入 zh.yaml 和 en.yaml
- [ ] 运行 `tests/_check_locale_keys.py` 验证通过
