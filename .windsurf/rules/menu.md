# 菜单规则

## 一、操作菜单固定序号（硬性要求）

所有软件进入操作菜单后，前 3 项**固定不变**，序号不可更改：

```
(1) 📥 安装
(2) 📤 卸载
(3) 🔼 升级
```

扩展操作从 (4) 开始按序追加：

```
(4) 🔧 诊断        ← 仅部分工具有
(5) ...             ← 未来扩展
(0) 🔙 返回
```

### 规则细节

- **序号 1-3 固定**：安装=1、卸载=2、升级=3，所有工具统一，用户养成习惯
- **扩展从 4 开始**：不同工具的扩展项可以不同，但序号连续递增
- **无分隔标题**：不显示「── 通用 ──」「── 扩展 ──」等分隔文字
- **不支持则灰显**：如 WireGuard 不支持升级，(3) 升级 仍然显示但灰色不可选，保持序号一致
- **0 始终为返回**：所有菜单层级统一

## 二、菜单层级结构

### 主菜单 → 软件管理 → 分类

```
OpsKit 主菜单
  └─ 📦 软件管理
       ├─ 🔍 搜索软件
       ├─ 🔨 开发工具
       │    ├─ Node.js
       │    └─ Python
       └─ 💻 运维工具
            ├─ Docker
            ├─ Nginx
            └─ WireGuard        ← 有子菜单的工具
                 ├─ 🌍 公网服务端
                 └─ 🏠 内网客户端
```

### 子菜单工具规则

- 含子菜单的工具（如 WireGuard）：点击后先展示子菜单选择角色，再进入操作菜单
- 无子菜单的工具（如 Docker）：点击后直接进入操作菜单

### 操作菜单进入后

- 标准 Recipe（Docker / Nginx）：选择安装 → 选版本 → 确认 → 执行
- 向导型 Recipe（WireGuard）：选择安装 → 选模式（快速/自定义）→ 填参数 → 确认 → 执行

## 三、菜单渲染规范

- 每个菜单项格式：`(序号) 图标 名称`
- 图标来自 `core/themes/catppuccin.yaml` 的 icon 映射
- 名称来自 `core/locale/zh.yaml` 或 `en.yaml` 的 i18n key
- 面包屑路径显示在菜单顶部（Powerline 风格色块）
- 已安装软件在分类列表中显示版本号（灰色）

## 四、快捷键规范（全平台 + 全插件强制）

- 数字键 `1-9`：选择菜单项（单键无需回车）
- `0`：返回上一级
- `ESC`：**唯一**的取消 / 返回键，抛出 UserCancel，由调用方决定返回还是退出
- `Ctrl+C` / `Ctrl+D`：**唯一**的终止键，抛出 UserExit 正常退出程序
- 方向键 / Home / End / PgUp / PgDn / Delete 等导航键**必须被忽略**：
  它们是 ESC 前缀序列（Unix `ESC [ ...` / `ESC O ...`）或双字符序列
  （Windows `\x00` / `\xe0` 前缀），不得触发取消、返回或写入输入缓冲；
  菜单与文本输入必须吞掉整个序列（core/prompt.py 的 `_read_key_seq` /
  `_read_line` 已统一实现）
- 插件禁止自行调用 `msvcrt` / `termios` / `input()` 读键，一律使用
  SDK 交互原语（select / multi_select / paged_select / confirm /
  text_input / pause），保证按键行为全平台一致
- Token / SecretKey / 密码等敏感输入必须 `text_input(..., secret=True)`：
  回显 `*`，默认值提示自动脱敏（只露首尾各 4 位）
