# 025 - Golang 安装验证改用 go version 子命令

## 变更

1. 修复 Golang 安装验证阶段：`_verify_runnable` 的版本参数改为可配置 `_version_args`，GoRecipe 使用 `("version",)`

## 背景

`VersionedTarballRecipe._verify_runnable` 固定运行 `<exe> --version`，但 go 不支持
`--version`（返回码 2，stderr 提示 `Use "go help <topic>"`），导致安装成功后
「验证安装」阶段报「安装后 go version 检测失败」。

## 修复方式

- `software/_shared/versioned_recipe.py`：新增类属性
  `_version_args: ClassVar[tuple[str, ...]] = ("--version",)`，验证命令改为
  `[exe, *self._version_args]`，其余 recipe 行为不变。
- `software/recipes/golang/recipe.py`：`_version_args = ("version",)`。

## 测试

- 新增 `tests/test_software.py::test_golang_verify_uses_version_subcommand`：
  伪造只接受 `version` 子命令的 go 可执行文件，验证通过。
- 全量：570 passed, 11 skipped。
